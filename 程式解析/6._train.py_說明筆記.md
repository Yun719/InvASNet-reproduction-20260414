## 二、執行順序逐段解析

### 2.1 全域區域（進入 main 之前）

```python
steps_per_epoch = 500
```

這是什麼？ 每個 Epoch 最多處理幾個 batch。DataLoader 可能有幾千筆， 但這裡人為限制每輪只跑 500 步，讓 Epoch 的時間可控。 ( 這個值應該要自動產生，而非手動設置，這是可修改點之一)

>容易誤解的地方：這不代表「資料集只有 500 筆」， 而是每輪只取資料集的前 500 個 batch，然後重新洗牌再進下一輪。

>修改點：如果資料量小（例如只有 100 首歌），可以把這個值調小， 或直接刪掉這個限制，讓它跑完整個 DataLoader。

### 2.2 Phase 1：環境與模型初始化

#### 裝置選擇

```python
device = torch.device("cuda:0" if torch.cuda.is_available() else "cpu")
```
PyTorch 的 Tensor 和模型必須在「同一個裝置」才能運算。 這行讓程式自動選 GPU（有的話）或 CPU，不需要手動改程式碼。

#### 模型建立與初始化

```python
net = Model().to(device)
init_model(net)
```

兩行各做一件事：

1. `Model()` — 建立模型物件，所有參數都在 CPU 記憶體

2. `.to(device)` — 把模型（含所有參數）複製到 GPU 記憶體

3. `init_model(net)` — 用 config 的 init_scale 做自訂初始化（覆蓋 PyTorch 預設值）；關於 `init_model` 的解釋請見 [model.py_說明筆記](5._model.py_說明筆記.md)


> 注意：`init_model` 必須在 `.to(device)` 之後呼叫， 因為 `init_model` 裡用了 `param.device`，需要知道實際裝置。

#### DataParallel（多 GPU）

```python
if use_cuda and len(getattr(c, "device_ids", [])) > 1:
    net = torch.nn.DataParallel(net, device_ids=c.device_ids)
```

只有 config.py 的 device_ids 有多個 GPU 才啟用。 DataParallel 會自動把 batch 切成多份分送到不同 GPU，結果合回來。

> ⚠️ 需注意點：啟用 DataParallel 後，`net.state_dict()` 的 key 前面會多 `module.`， `load_ckpt` 裡有用 `k.replace('module.', '')` 來解決這個問題。

#### Optimizer：Adam

```python
optimizer = optim.Adam(
    params_trainable,
    lr=c.lr,           # 學習率，約 3.16e-5
    betas=(0.5, 0.999),
    eps=1e-6,
    weight_decay=1e-5,
)
```

Adam 是什麼？ Adam 是目前最主流的優化器。它會為每個參數自動調整「有效學習率」， 讓訓練比純 SGD 更快收斂、更穩定。

| 參數             | 預設值          | 作用                         |
| -------------- | ------------ | -------------------------- |
| `lr`           | 3.16e-5      | 每次更新參數的基礎步伐大小              |
| `betas`        | (0.5, 0.999) | 動量的衰減係數（前者控制梯度方向，後者控制梯度大小） |
| `eps`          | 1e-6         | 防止除以零的小數                   |
| `weight_decay` | 1e-5         | L2 正則化，防止過擬合（讓參數不要變太大）     |

> 最常調的地方：lr（學習率）。Loss 不下降 → 試試調小 10 倍； 訓練太慢 → 試試調大 2~3 倍。lr 在 `config.py` 中

#### 為什麼只傳 params_trainable？

```python
params_trainable = list(filter(lambda p: p.requires_grad, net.parameters()))
```

只讓「允許更新的參數」加入 optimizer。 如果未來你冷凍（freeze）某些層，它們的 requires_grad=False， 就不會被更新，也不會浪費計算資源算梯度。

#### Scheduler：StepLR

```python
scheduler = optim.lr_scheduler.StepLR(
    optimizer,
    step_size=c.weight_step,  # 每 150 個 epoch
    gamma=c.gamma,            # 學習率乘以 0.5
)
```

Scheduler 是什麼？ 訓練過程中自動降低學習率的機制。

> weight_step 可以在 `config` 中設定

舉例：

```python
Epoch 1~150:   lr = 3.16e-5
Epoch 151~300: lr = 1.58e-5  （× 0.5）
Epoch 301~450: lr = 0.79e-5  （× 0.5）
...
```

為什麼要降低學習率？ 訓練初期需要大步伐快速接近最佳解， 後期需要小步伐精細調整，避免在最佳解附近來回振盪。

> ⚠️ 需注意點：`scheduler.step()` 要放在 epoch 迴圈的最後面， 每個 epoch 結束才更新一次學習率。如果你不小心把它放進 batch 迴圈裡， 學習率會降低過快，導致訓練效果變差。

### 2.3 Phase 2：載入 Checkpoint（Resume 機制）

```python
if getattr(c, "tain_next", False):   # 注意：typo，是 "tain" 不是 "train"
    load_ckpt(net, optimizer, ckpt_path, map_location=device)
```

> 因為 `config.py` 裡把 train_next 拼成 tain_next ，所以這裡沿用拼錯的版本  

`load_ckpt` 內部做了什麼？

```python
def load_ckpt(net, optimizer, path, map_location):
    state = torch.load(path, map_location=map_location)
    # 過濾掉 tmp_var（某些框架的暫存變數，不需要載入）
    net_state = {k: v for k, v in state["net"].items() if "tmp_var" not in k}
    net.load_state_dict(net_state, strict=False)  # strict=False：允許部分匹配
    optimizer.load_state_dict(state["opt"])        # 連 optimizer 狀態一起載入
```

`strict=False` 的意義：如果 checkpoint 的結構和現在的模型不完全一樣 （例如你新增了一層），也不會報錯，只載入能對應上的部分。

### 2.4 Phase 3：訓練迴圈（最核心的部分）

Tensor Shape 完整追蹤

以 `B=2, C=1, L=44160` 為例：

（ (B,C,L) = (batch_size, Channel, 取樣率) ）

這裡對應到程式中的 `for i_batch, batch in pbar:` 迴圈

```
cover.wav  →  (2, 1, 44160)   ← DataLoader 輸出
secret.wav →  (2, 1, 44160)

DWT(cover)  → cover_d  (2, 2, 22080)   ← 通道 ×2，長度 ÷2
DWT(secret) → secret_d (2, 2, 22080)

cat([cover_d, secret_d], dim=1) → x  (2, 4, 22080)   ← 4 通道

net(x, rev=False) → y  (2, 4, 22080)   ← 形狀不變

y.narrow(1, 0, 2)      → y_steg  (2, 2, 22080)   ← 前半（對應 cover）
y.narrow(1, 2, 2)      → y_z     (2, 2, 22080)   ← 後半（對應 secret）

IWT(y_steg) → steg  (2, 1, 44160)   ← 還原成波形

gauss_noise_like(y_z)  → z_rand  (2, 2, 22080)
cat([y_steg, z_rand])  → y_rev_in (2, 4, 22080)

net(y_rev_in, rev=True) → x_hat  (2, 4, 22080)

x_hat.narrow(1, 2, 2)  → secret_hat_d (2, 2, 22080)
IWT(secret_hat_d)       → secret_hat   (2, 1, 44160)   ← 還原出的秘密
```

> cat：組合；narrow：切割

| 參數             | 說明                                                   |
| -------------- | ---------------------------------------------------- |
| cover  | 載體音樂 |
| secret | 秘密語音 |
| cover_d| DWT 後的載體音樂 |
| secret_d | DWT 後的載秘密音樂 |
| x | 使用 cat 把 cover_d 和 secret_d 組合成 x，cover 在前半段；secret 在後半段|
| y | x 經過神經網路正向隱藏處理後的音訊 |
| y_steg | 藏秘密音訊的音樂 |
| y_z | 會被丟棄的噪音 |
| steg   | 藏了秘密的音訊|
| z_rand | 程式產生的高斯雜訊 |
| y_rev_in | 把 steg 和 z_rand 組合在一起|
| x_hat | y_rev_in 經過神經網路反向還原處理後的音訊 |
| secret_hat_d  | 後半段語音的部分 |
|  secret_hat | secret_hat_d 做反 DWT 後得到的還原後秘密語音|


為什麼用隨機雜訊取代 y_z？

```python
z_rand = gauss_noise_like(y_z)  # 高斯雜訊
y_rev_in = torch.cat([y_steg, z_rand], dim=1) # 把這兩個合在一起
```

實際使用時（推論階段），使用者只有 stego 音樂， 完全不知道 y_z 的值是什麼。 訓練時故意模擬這個情境，讓模型學會： 「即使後半是未知的雜訊，也要能從前半（stego）把 secret 挖出來。」

這讓 secret 的資訊被「強制」編碼進 stego 的前半部分（y_steg）， 而不能藏在 y_z 裡（因為推論時 y_z 會被換掉）。

### 2.5 三個 Loss 函數詳解

```python
# Loss 1: 偽裝品質（Guide Loss）
g_loss = mse_loss_mean(steg, cover)

# Loss 2: 還原品質（Reconstruction Loss）
r_loss = mse_loss_mean(secret_hat, secret)

# Loss 3: 低頻約束（Low-frequency Loss）
steg_low  = y_steg.narrow(1, 0, channels_in)   # stego 的低頻子帶
cover_low = cover_d.narrow(1, 0, channels_in)   # cover 的低頻子帶
l_loss = mse_loss_mean(steg_low, cover_low)

# 加權合計
total = 5 * r_loss + 3 * g_loss + 1 * l_loss
```

| Loss     | 目標                  | 權重    | 設計理念           |
| -------- | ------------------- | ----- | -------------- |
| `g_loss` | stego 要聽起來像 cover   | 3     | 隱寫不可察覺性        |
| `r_loss` | 還原的 secret 要準確      | 5（最高） | 這是最核心目標        |
| `l_loss` | stego 低頻要和 cover 一樣 | 1     | 人耳對低頻最敏感，保護主旋律 |

>註：這邊的權重由 `config.py` 的 `lamda_reconstruction` 、 `lamda_guide` 、 `lamda_low_frequency` 定義

MSE Loss 是什麼？

MSE = 平均( (預測值 - 真實值)² )
值越小代表兩個張量越接近。MSE 對大誤差的懲罰比小誤差大很多（平方的效果）。

> 修改 Loss 的入口：直接改這段的公式和權重（lam_r, lam_g, lam_l）。 例如想用 L1 Loss（對雜訊更穩健），可以把 mse_loss_mean 換成 nn.L1Loss(reduction="mean")(a, b)。

### 2.6 Backward / Gradient / 參數更新

```python
optimizer.zero_grad(set_to_none=True)   # Step A
total.backward()                          # Step B
torch.nn.utils.clip_grad_norm_(net.parameters(), max_norm=1.0)  # Step C
optimizer.step()                          # Step D
```

#### **四步的真實意義：**

**Step A** — 清除梯度 PyTorch 預設會把每次計算的梯度「累加」在 param.grad 上。 如果不清除，每個 batch 的梯度都會疊加在一起，方向完全錯誤。 `set_to_none=True` 比 `zero_grad()` 更省記憶體（直接把 grad 設為 None）。

**Step B** — 反向傳播 `total.backward()` 會對計算圖做鏈式法則（Chain Rule）， 自動算出「每個參數對 total loss 的貢獻（梯度）」， 存在各參數的 `.grad` 屬性裡。

```
梯度 = ∂ total_loss / ∂ param
梯度越大 → 這個參數對 loss 的影響越大 → 需要更大幅度地更新
```
**Step C** — 梯度裁剪（Gradient Clipping）

```python
torch.nn.utils.clip_grad_norm_(net.parameters(), max_norm=1.0)
```

這行限制所有參數的梯度「向量的總長度」不超過 1.0。

為什麼需要這個？ 可逆神經網路裡的 exp() 運算在某些邊界條件下會產生極大的梯度， 一次更新就把參數推到很遠的地方，訓練崩潰（梯度爆炸）。 裁剪梯度相當於「把更新步伐的上限鎖住」。

>修改點：如果訓練很穩定，可以把 max_norm 調大（如 5.0）讓模型收斂更快； 如果出現 NaN，可以調小（如 0.5）。

**Step D** — 參數更新

param = param - lr × Adam調整後的梯度

Adam 在純梯度下降的基礎上，還考慮了過去梯度的「動量」和「自適應縮放」， 讓每個參數都有最適合自己的有效學習率。

### 2.7 Validation（驗證）

```python
if val_freq > 0 and (i_epoch % val_freq == 0):
    net.eval()
    with torch.no_grad():
        ...
```

兩個關鍵點：

`net.eval()`：切換到推論模式。影響有 BatchNorm 和 Dropout 的行為， 本模型沒有這些層，但這是好習慣必須加。

`torch.no_grad()`：告訴 PyTorch 不要記錄計算圖， 節省大量記憶體（validation 不需要算梯度）。

>⚠️ 注意：因為缺乏驗證資料集，目前 val_freq = 0（在 config.py），所以驗證會被被跳過，如果想開啟，把 val_freq 改成正整數（例如 10）。

### 2.8 Checkpoint 存檔策略

```python
# 每個 epoch 存一次（目前 SAVE_freq = 1）
torch.save({"opt": optimizer.state_dict(), "net": net.state_dict()}, save_path)

# 訓練完成後存最終版
torch.save(..., "model.pt")

# 崩潰時緊急存檔
torch.save(..., "model_ABORT.pt")
```

為什麼同時存 net 和 opt？ 只存 net 的話，下次載入繼續訓練時，optimizer 的動量狀態（moment）會歸零， 相當於重新從冷啟動開始，前幾個 epoch 學習率會不穩定。 一起存 optimizer 狀態就能無縫接續。

>註：存檔頻率寫在 `config.py` 中的 `SAVE_freq`

## 七、初學者最容易卡住的觀念

**Q1：`backward()` 到底在做什麼？**

> 沿著「計算圖」用微分鏈式法則，算出每個參數的梯度。 你不需要手動寫任何微分公式，PyTorch 自動記錄了所有運算，`backward()` 一呼叫就全算好。

**Q2：為什麼 `zero_grad()` 一定要在 `backward()` 之前？**

> PyTorch 預設梯度是「累加」而不是「覆寫」， 不清除的話，這次的梯度會加上上次的梯度，方向完全錯誤。

**Q3：`net.train()` 和 `net.eval()` 差在哪？**

> 主要影響 BatchNorm 和 Dropout 的行為。 train 模式：BatchNorm 用當前 batch 統計；Dropout 隨機丟棄。 eval 模式：BatchNorm 用全局統計；Dropout 關閉。 本模型沒有這兩個，但切換是好習慣。

**Q4：`scheduler.step()` 為什麼放在迴圈最後？**

> Scheduler 記錄「第幾個 epoch」來決定何時降低學習率。 放在 epoch 迴圈結尾（不是 batch 迴圈裡）才能正確地「每個 epoch 更新一次」。

**Q5：`.item()` 是什麼？**

> 把一個只有一個數字的 Tensor 取出成 Python 的 `float`。 例如 `total.item()` → `0.0043`（普通數字）。 用 `float()` 可以順便切斷梯度追蹤，避免記憶體洩漏。
