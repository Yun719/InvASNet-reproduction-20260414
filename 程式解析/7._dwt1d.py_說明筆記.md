## 二、什麼是 DWT（離散小波轉換）？

### 2.1 白話解釋

想像一首歌的聲音波形像一條「高低起伏的線」。 DWT 把這條線分成兩條：

| 子帶           | 代表什麼          | 人耳感受           |
| ------------ | ------------- | -------------- |
| **低頻（Low）**  | 整體輪廓、主旋律、低沉部分 | 最敏感，改了馬上聽得出來   |
| **高頻（High）** | 細節、音色變化、高頻雜訊  | 較不敏感，微小改動不容易察覺 |

分成兩條之後，每條的長度各是原來的一半：

```
原始：[s0, s1, s2, s3, s4, s5, ...]  長度 L
         ↓
低頻：[(s0+s1)/2, (s2+s3)/2, ...]    長度 L/2
高頻：[(s0-s1)/2, (s2-s3)/2, ...]    長度 L/2
```

### 2.2 為什麼要先做 DWT，不直接餵原始波形進模型？

| 理由         | 說明                                    |
| ---------- | ------------------------------------- |
| **幫助隱寫**   | 網路可以把秘密主要藏在高頻子帶，對低頻（主旋律）的干擾最小         |
| **降低序列長度** | L → L/2，計算量減半，訓練更快                    |
| **資訊分離**   | 低頻和高頻被明確分開，loss 可以針對低頻做額外約束（`l_loss`） |
| **可逆性**    | Haar DWT 有完美的逆運算（IWT），資訊零損失           |

## 三、完整程式碼標注版

### DWT1D（正向：時域 → 頻率域）

```python
class DWT1D(nn.Module):
    """
    1D Haar DWT for audio.
    Input:  x  shape (B, C, L)  where L is even
    Output: y  shape (B, 2C, L/2)  = concat(low, high) along channel dim

    low  = (x_even + x_odd) / 2
    high = (x_even - x_odd) / 2
    """
    def __init__(self):
        super().__init__()
        # 注意：這裡沒有任何 nn.Parameter！
        # DWT 是純數學運算，沒有可訓練的權重，不需要學習。

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        if x.dim() != 3:
            raise ValueError(f"DWT1D expects (B,C,L), got {tuple(x.shape)}")
        B, C, L = x.shape
        if L % 2 != 0:
            raise ValueError(f"Length L must be even for Haar DWT, got L={L}")

        # 把波形按「偶數位置」和「奇數位置」拆開
        x_even = x[:, :, 0::2]   # 取 index 0, 2, 4, 6, ...  → shape (B, C, L/2)
        x_odd  = x[:, :, 1::2]   # 取 index 1, 3, 5, 7, ...  → shape (B, C, L/2)

        # Haar 小波的核心公式
        low  = (x_even + x_odd) * 0.5   # 相鄰兩點的平均 → 低頻
        high = (x_even - x_odd) * 0.5   # 相鄰兩點的差值 → 高頻

        # 沿通道維度（dim=1）拼接，低頻在前、高頻在後
        return torch.cat([low, high], dim=1)
        # 輸出 shape: (B, 2C, L/2)
```
### IWT1D（反向：頻率域 → 時域）

```python
class IWT1D(nn.Module):
    """
    Inverse 1D Haar DWT.
    Input:  y shape (B, 2C, L/2)
    Output: x shape (B, C, L)

    x_even = low + high
    x_odd  = low - high
    """
    def __init__(self):
        super().__init__()
        # 同樣：沒有可訓練參數

    def forward(self, y: torch.Tensor) -> torch.Tensor:
        if y.dim() != 3:
            raise ValueError(f"IWT1D expects (B,2C,L/2), got {tuple(y.shape)}")
        B, CC, Lh = y.shape
        if CC % 2 != 0:
            raise ValueError(f"Channel dim must be even (2C), got {CC}")

        C = CC // 2
        low  = y[:, :C, :]    # 前 C 個通道 = 低頻子帶
        high = y[:, C:, :]    # 後 C 個通道 = 高頻子帶

        # 逆 Haar 公式（完美還原）
        x_even = low + high   # 偶數位置的採樣點
        x_odd  = low - high   # 奇數位置的採樣點

        # 把偶數和奇數位置「交錯插入」，還原成長度 2*Lh 的序列
        L = Lh * 2
        x = torch.empty((B, C, L), device=y.device, dtype=y.dtype)
        x[:, :, 0::2] = x_even   # 放回偶數位置
        x[:, :, 1::2] = x_odd    # 放回奇數位置

        return x
        # 輸出 shape: (B, C, L)
```

## 四、數學原理圖解

### 4.1 為什麼這個公式能完美還原？

```python
已知：
  low  = (x_even + x_odd) / 2
  high = (x_even - x_odd) / 2

推導還原公式：
  low + high = (x_even + x_odd)/2 + (x_even - x_odd)/2
             = (2 × x_even) / 2
             = x_even  ✅

  low - high = (x_even + x_odd)/2 - (x_even - x_odd)/2
             = (2 × x_odd) / 2
             = x_odd  ✅
```

加法和減法互為逆運算，所以資訊零損失，可以100%還原。

### 4.2 用數字範例驗證

```
原始波形片段（L=4）：
  x = [0.8, 0.6, 0.3, 0.7]

  x_even = [0.8, 0.3]   (index 0, 2)
  x_odd  = [0.6, 0.7]   (index 1, 3)

DWT：
  low  = [(0.8+0.6)/2, (0.3+0.7)/2] = [0.70, 0.50]
  high = [(0.8-0.6)/2, (0.3-0.7)/2] = [0.10, -0.20]

IWT（驗證還原）：
  x_even = low + high = [0.70+0.10, 0.50+(-0.20)] = [0.80, 0.30] ✅
  x_odd  = low - high = [0.70-0.10, 0.50-(-0.20)] = [0.60, 0.70] ✅

交錯插回：
  x = [0.80, 0.60, 0.30, 0.70]  ← 完全還原！✅
```

## 六、DWT 沒有可訓練參數——這很重要！

```python
def __init__(self):
    super().__init__()
    # 空的！沒有 nn.Linear, nn.Conv1d, nn.Parameter...
```

這代表：
- DWT/IWT 是**固定的數學函數**，不會被訓練更新
- 它的行為永遠一致，不會隨訓練改變
- 它的輸入輸出關係是 100% 可預測的

但是，`torch.no_grad()` 裡仍然可以正常呼叫它，
因為它不需要梯度（沒有參數需要更新）。

## 七、在 train.py 的使用方式

```python
# Phase 1：初始化（只建立一次，不需要 .to(device) 因為沒有參數）
dwt = DWT1D()
iwt = IWT1D()

# 每個 batch 的使用順序：
cover_d = dwt(cover)     # 音訊 → 頻率域
secret_d = dwt(secret)   # 音訊 → 頻率域

x = torch.cat([cover_d, secret_d], dim=1)

y = net(x)               # 模型在頻率域操作

y_steg = y.narrow(1, 0, 2)
steg = iwt(y_steg)       # 頻率域 → 音訊（輸出的偽裝音樂）

secret_hat_d = x_hat.narrow(1, 2, 2)
secret_hat = iwt(secret_hat_d)  # 頻率域 → 音訊（還原的秘密）
```

> ⚠️ **注意**：`dwt` 不需要 `.to(device)`，因為它沒有參數。
> 但它操作的 tensor 必須已經在正確的裝置上。

## 八、常見問題與 Bug 點

### 8.1 長度必須是偶數

```python
if L % 2 != 0:
    raise ValueError(f"Length L must be even, got L={L}")
```

`config.py` 裡 `segment_length = 44160`，44160 是偶數 ✅。
但如果你改了 `segment_length`，一定要確認新值是偶數，否則 DWT 會報錯。

> 🔧 保險做法：在 `datasets.py` 裡裁切時，用 `(L // 2) * 2` 確保偶數：
> ```python
> valid_len = (waveform.shape[1] // 2) * 2
> waveform = waveform[:, :valid_len]
> ```

### 8.2 IWT 的輸入通道數必須是偶數

```python
if CC % 2 != 0:
    raise ValueError(f"Channel dim must be even (2C), got {CC}")
```

傳入 IWT 的張量，通道數必須是 2 的倍數（因為前半是 low、後半是 high）。
如果你在 `y_steg` 的 narrow 切的範圍有誤，這裡會報錯。

### 8.3 DWT/IWT 不需要 eval() 切換

因為沒有 BatchNorm 或 Dropout，`net.eval()` 不影響 DWT/IWT 的行為。
它永遠是同樣的數學公式。

## 九、如何修改或替換

### 9.1 換成多層 DWT（多解析度分析）

目前只做一層 DWT（L → L/2）。
如果想要更細緻的頻率分析，可以對低頻子帶再做一次 DWT：

```python
cover_d1 = dwt(cover)                    # L → L/2，產生 low1 + high1
low1 = cover_d1[:, :channels_in, :]      # 取低頻子帶
cover_d2 = dwt(low1.unsqueeze(-1)...)    # 再對低頻做一次 DWT
# ⚠️ 這樣模型的輸入通道數會改變，需要對應修改 INV_block 的 in_1, in_2
```

### 9.2 換成 STFT（短時傅立葉變換）

如果想用頻譜圖（spectrogram）代替 DWT：

```python
import torch
stft = torch.stft(cover.squeeze(1), n_fft=512, hop_length=256, return_complex=True)
# ⚠️ STFT 輸出是複數，需要分實部和虛部，且不像 Haar DWT 那樣有簡單的逆運算
# ⚠️ 這樣改動很大，會影響模型輸入輸出的 shape
```

### 9.3 驗證可逆性（Debug 工具）

```python
import torch
from modules.dwt1d import DWT1D, IWT1D

dwt = DWT1D()
iwt = IWT1D()

x = torch.randn(1, 1, 44160)   # 隨機模擬一段音訊
x_reconstructed = iwt(dwt(x))  # DWT 然後 IWT

error = (x - x_reconstructed).abs().max()
print(f"最大還原誤差: {error:.2e}")
# 正常結果應該是 < 1e-6（浮點數精度誤差）
```
